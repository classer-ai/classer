TRANSACTIONAL EMAILS
====================

All emails are sent via AWS SES from notifications@classer.ai
Source code: api/email_service.py


TIER 1 — CRITICAL (cannot be disabled by user)
===============================================

1. TOPUP CONFIRMATION
---------------------
Type:    topup_confirmation
Subject: Payment confirmed: $X.XX added to Classer
Trigger: Stripe webhook after successful payment
Dedup:   None (always sent)

Variables: amount_cents, new_balance_cents

HTML:

    <h2>Payment Confirmed</h2>
    <p>Your payment of <strong>$X.XX</strong> was successful.</p>
    <p>New balance: <strong>$X.XX</strong></p>
    <p><a href="https://classer.ai/billing">View billing</a></p>

Plain text:

    Payment Confirmed

    Your payment of $X.XX was successful.
    New balance: $X.XX

    View billing: https://classer.ai/billing


2. PAYMENT FAILED
-----------------
Type:    payment_failed
Subject: Payment failed - Action required
Trigger: Stripe webhook on failed charge
Dedup:   None (always sent)

Variables: error_message

HTML:

    <h2>Payment Failed</h2>
    <p>Your payment was unsuccessful.</p>
    <p><strong>Reason:</strong> {error_message}</p>
    <p><a href="https://classer.ai/billing">Update payment method</a></p>

Plain text:

    Payment Failed

    Your payment was unsuccessful.
    Reason: {error_message}

    Update payment method: https://classer.ai/billing


3. HARD LIMIT REACHED
---------------------
Type:    hard_limit_reached
Subject: API paused - Spending limit reached
Trigger: Billing service when hard spend limit is hit
Dedup:   Once per calendar month

Variables: limit_cents, spend_cents

HTML:

    <h2>Spending Limit Reached</h2>
    <p>Your API is paused because you've reached your spending limit.</p>
    <p>Limit: $X.XX, Spent: $X.XX</p>
    <p><a href="https://classer.ai/settings">Increase limit</a></p>

Plain text:

    Spending Limit Reached

    Your API is paused.
    Limit: $X.XX
    Spent: $X.XX

    Increase limit: https://classer.ai/settings


TIER 2 — OPTIONAL (user can toggle off in Settings > Email Notifications)
=========================================================================

4. SOFT LIMIT WARNING
---------------------
Type:       soft_limit_warning
Subject:    Spending alert - Soft limit reached
Trigger:    Billing service when soft spend limit is hit
Dedup:      Once per calendar month
Preference: soft_limit_alerts (default: on)

Variables: limit_cents, spend_cents

HTML:

    <h2>Soft Limit Warning</h2>
    <p>You've reached your soft spending limit.</p>
    <p>Limit: $X.XX, Spent: $X.XX</p>
    <p>Your API will continue to work. <a href="https://classer.ai/settings">Adjust limits</a></p>

Plain text:

    Soft Limit Warning

    Limit: $X.XX, Spent: $X.XX

    Your API will continue to work.
    Adjust limits: https://classer.ai/settings


5. WELCOME
----------
Type:       welcome
Subject:    Welcome to Classer!
Trigger:    New user signup (auth event)
Dedup:      Once ever
Preference: welcome_emails (default: on)

Variables: name

HTML:

    <h2>Welcome, {name}!</h2>
    <p>Your account is ready with 5M free tokens/month.</p>
    <p>Get started:</p>
    <ol>
      <li><a href="https://classer.ai/api-keys">Create an API key</a></li>
      <li><a href="https://docs.classer.ai">Read the docs</a></li>
    </ol>

Plain text:

    Welcome, {name}!

    Your account is ready with 5M free tokens/month.

    1. Create API key: https://classer.ai/api-keys
    2. Docs: https://docs.classer.ai


6. LOW BALANCE WARNING
----------------------
Type:       low_balance_warning
Subject:    Low balance warning
Trigger:    Billing service when balance drops below $5
Dedup:      Once per calendar month
Preference: low_balance_alerts (default: on)

Variables: balance_cents

HTML:

    <h2>Low Balance</h2>
    <p>Your balance is <strong>$X.XX</strong>.</p>
    <p><a href="https://classer.ai/billing">Add credits</a> or enable auto top-up to avoid service interruption.</p>

Plain text:

    Low Balance

    Your balance is $X.XX.

    Add credits: https://classer.ai/billing


TEMPLATE VARIABLES
==================

All _cents values are integers divided by 100 for display (e.g. 1050 -> $10.50).

  Topup confirmation:  amount_cents, new_balance_cents
  Payment failed:      error_message
  Hard limit reached:  limit_cents, spend_cents
  Soft limit warning:  limit_cents, spend_cents
  Welcome:             name
  Low balance warning: balance_cents


DELIVERY TRACKING
=================

SES delivery/bounce/complaint events are received via SNS webhook at /api/webhooks/ses.
All emails are logged in the email_logs table with status flow:
  queued -> sent -> delivered / bounced / complained / failed
