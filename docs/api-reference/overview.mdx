## Base URL

```
https://api.classer.ai
```

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | [`/v1/classify`](/api-reference/classify) | Single-label classification |
| POST | [`/v1/tag`](/api-reference/tag) | Multi-label tagging |
| POST | `/v1/binary` | Binary yes/no scoring |
| POST | `/v1/classify/batch` | Batch single-label classification (up to 1000 texts) |
| POST | `/v1/tag/batch` | Batch multi-label tagging (up to 1000 texts) |
| POST | `/v1/batch` | Create async batch job |
| POST | `/v1/batch/{job_id}/start` | Start async batch processing |
| GET | `/v1/batch/{job_id}` | Poll batch job status |
| GET | `/v1/batch/{job_id}/results` | Download batch results |
| DELETE | `/v1/batch/{job_id}` | Delete batch job |
| GET | `/v1/classifiers` | List saved classifiers |
| POST | `/v1/classifiers` | Create/save a classifier |
| GET | `/v1/classifiers/{name}` | Get classifier details |
| GET | `/v1/keys` | List API keys |
| POST | `/v1/keys` | Create API key |
| GET | `/v1/balance` | Get balance and spending |
| GET | `/v1/usage` | Get usage statistics |
| GET | `/v1/logs` | Get request logs |

## Authentication

All requests require a Bearer token in the `Authorization` header:

```bash
Authorization: Bearer your-api-key
```

See [Authentication](/authentication) for details.

## Request format

All endpoints accept JSON with `Content-Type: application/json`.

## Response format

Classification responses include:

| Field | Type | Description |
|-------|------|-------------|
| `tokens` | number | Total tokens used |
| `latency_ms` | number | Processing time in milliseconds |
| `cached` | boolean | Whether the response was served from cache |
| `public` | boolean | Present when using public (unauthenticated) mode |

Plus endpoint-specific fields (`label`, `confidence`, `labels`, etc.).

### Example response

```json
{
  "label": "technical",
  "confidence": 0.9423,
  "tokens": 142,
  "latency_ms": 87,
  "cached": false
}
```

## Error responses

```json
{
  "code": "validation_error",
  "message": "Description of what went wrong",
  "field": "labels",
  "details": "Additional context (optional)"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `code` | string | Machine-readable error code |
| `message` | string | Human-readable error message |
| `field` | string | The request field that caused the error (optional) |
| `details` | string | Additional context (optional) |

| Status | Description |
|--------|-------------|
| `400` | Bad request — invalid parameters |
| `401` | Unauthorized — missing or invalid API key |
| `402` | Insufficient balance |
| `422` | Validation error — check request body |
| `429` | Rate limit exceeded |
| `500` | Internal server error |

## Speed tiers

The `/v1/classify` and `/v1/tag` endpoints accept a `speed` parameter that controls latency and pricing.

| Speed | Latency | Per 1M tokens |
|-------|---------|---------------|
| `fast` (default) | <200ms | $0.60 |
| `standard` | <1s | $0.20 |

The Fast Batch endpoints (`/v1/classify/batch`, `/v1/tag/batch`) are billed at **$0.08/1M tokens** automatically — no `speed` parameter needed.

Every account gets **10M free tokens/month** on the standard tier.

## Rate limits

Rate limits depend on the tier configured for your API key.

| Tier | Requests/min | Tokens/min | Concurrent |
|------|-------------|------------|------------|
| Public | 30 | 150,000 | 2 |
| Developer | 60 | 500,000 | 5 |
| Production | 1,000 | 10,000,000 | 50 |
| Enterprise | Custom | Custom | Custom |
